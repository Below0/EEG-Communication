## Naming Rule
#### issue naming rule
* `<tag>:<title>`
* tag
  * DEV(develop)
  * DES(design)
  * INF(information)
* ex) `DEV:Project Setup`
#### branch naming rule
* master :  완성된 프로젝트
* dev : 개발 중인 프로젝트
* `EC<issue number>_<summary>`
  * ex) `EC15_fix_bluetooth_module`
#### pull request naming rule
* `EC<issue number>: summary`
* `EC15: fix error`
